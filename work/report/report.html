Give a brief explanation of how the browser, the HTTP protocol and the web server interacts.

(a) the browser resolves the host name of the server from the URL; (b) The browser uses the domain name resolution mechanism to translate the host name of the server into the IP address of the server; (how to use IP instead of the domain name directly, steps a and b can be omitted) (c) the browser resolves the port number (if any) from the URL; (d) The browser establishes a TCP connection with the Web server through the IP address and port number; (e) The browser sends an HTTP request message to the server; (f) the server sends back an HTTP response message to the browser; (g) The connection is closed and the browser displays the document.

Can you compare and relate the JavaScript language to any other language you know?

I want to compare the JS language to jave. 1) Object-based and object-oriented: Java is a true object-oriented language. Even if you are developing a simple program, you must design the object; JavaScript is a scripting language that can be used to make network-independent and interact with users Complex software. It is an object-based and event-driven programming language. So it provides very rich internal objects for designers to use; 2) Interpretation and compilation: Java's source code must be compiled before execution; JavaScript is an interpreted programming language, and its source code does not need to be compiled and interpreted by the browser; 3) Strongly typed variables and weakly typed variables: Java uses strongly typed variable checking, that is, all variables must be declared before compilation; variable declarations in JavaScript use their weak types. That is, the variable does not need to be declared before use, but the interpreter checks its data type at runtime; 4) The code format is different.

I would definitely recommend node.js for several reasons. 1. Concurrent connection, Asynchronous, event-driven model The asynchronous mechanism and event-driven process do not block the connection of new users, and do not need to maintain the connection of existing users. Based on this mechanism, in theory, there are user requests to connect in succession, and NodeJS can respond. Therefore, NodeJS can support higher concurrency than Java and PHP programs. Although maintaining an event queue also requires costs, because NodeJS is a single thread, events The longer the queue, the longer it takes to get a response, and the amount of concurrency will still be weak. NodeJS solves the problem of concurrent connections: change the way to connect to the server, each connection emits an event running in the NodeJS engine process and put it in the event queue, instead of generating a new OS for each connection Threads (and allocate some accompanying memory for them). 2. Non I/O blocking Java and PHP also have ways to implement parallel requests (child threads), but NodeJS does it naturally through callback functions (Callback) and asynchronous mechanisms.
